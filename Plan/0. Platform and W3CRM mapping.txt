2. W3CRM folder mapping (where USIS work lives)
-----------------------------------------------
• Page shells & new USIS pages: gulp\src\*.html (copy from nearest demo, then rename for USIS).
• Layout fragments (“partials”): gulp\src\elements\
  - index.html wires: preloader, nav-header, chatbox, header, deznav (sidebar).
  - USIS construction shell: elements\deznav-construction.html (MetisMenu #menu) with module links.
• Global / shared JS: gulp\src\assets\js\ (custom.js, deznav-init-construction.js; add usis-*.js as needed).
• Styles: gulp\src\assets\scss\ (layout\sidebar for nav chrome).
• Dev workflow: from W3CRM-v3.0-13_September_2025\gulp run npm install then npx gulp (BrowserSync + watch). Production: npx gulp build → dist\
• BrowserSync prints the real Local URL in the terminal (default start path is /construction/index.html). If port 3000 is already in use, BrowserSync uses the next free port (3001, 3002, …); bookmarking localhost:3000 while the server is on another port looks like “the site won’t load.” After npx gulp build, run npx gulp again if you preview via BrowserSync so ./.temp is regenerated. Pages that call Flask (e.g. HR dashboard) treat the shell as listening on ports **3000–3099** on the same hostname and send API traffic to **port 5000** on that host, unless overridden by `<meta name="usis-api-base">` or `window.USIS_API_BASE`.

4. AI UI strategy (Plan 12) — locked for W3CRM phase
----------------------------------------------------
• Pattern: Vanilla ES modules (or single IIFE bundle) under assets\js\, no React requirement.
• Event bus: implement aiReviewBus as assets\js\aiReviewBus.js (same responsibilities as the old .ts bus: subscribe/publish review events, provider mode).
• Chat UI: Bootstrap offcanvas or modal launched from header; messages rendered as HTML strings or lightweight template literals; streaming via fetch + ReadableStream or SSE from existing Flask /api/ai/* endpoints.
• Rationale: Matches W3CRM stack, avoids shipping React for the shell; Flask contracts in Plans stay valid. If a future phase introduces React islands, wrap only the chat widget.
